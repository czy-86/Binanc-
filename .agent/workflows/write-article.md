# 写文章/帖子工作流 (Reflexion 自动纠错车间)

当用户调用 `/write-article` 并提供素材或主题时，你必须**绝对**按照以下多轮内部逻辑链（Multi-step Workflow）进行后台作业。在完成前三步之前，**严禁**向用户输出最终文章。

## ⚙️ 第一阶段：前置确认（与用户对齐）
1. 询问用户需要的**内容体裁**（深度长文 还是 犀利短帖）。
2. 从素材中提取 1-2 个极具冲突感的**核心切入点**供用户选择。
   *（如果用户指令中已经明确了体裁和角度，可跳过此阶段直接进入作业）*

## ⚙️ 第二阶段：黑盒草稿生成（内部推演，不对外展示）
根据用户的素材和确定的切入点，调用对应的 `long_article_manual.md` 或 `short_post_manual.md`，并在你的内部思考沙盘（`<thought>` 块或静默执行）中生成初代草稿。
- **注意**：此时不要输出给用户看。

## ⚙️ 第三阶段：无情自审（红队判决）
必须在内部针对刚刚生成的草稿，逐条核对核心底层引擎 `web3_philosopher_core.md` 中的判点。如果不满足以下**任何一条**，必须对自己进行“严厉打回”并记录修改意见：
1. **反幻觉检查**：是否使用了“比特犬/算力狗狗、法拉利、高速公路”等已经被锁死的陈旧比喻？（如果有，打回！）
2. **反CX词汇禁令**：是否使用了“百倍 Alpha、财富密码、暴富、史诗级”等廉价喊单词汇？（如果有，必须改成冷酷的商业收税收割逻辑，打回！）
3. **情绪穿透（FOMO/FUD）**：开头是否有强烈的压迫感（FUD）？文章前30%是否制造了“高级 FOMO”（制造错过顶层逻辑/聪明钱布局的智商焦虑，绝不是低端喊单）？（如果变成了低级推销或干瘪科普，打回！）
4. **价值收敛**：宏大叙事最后有没有折叠到“代币必须要被消耗/质押”或“散户如何省钱/赚钱”上？（如果只谈技术不谈钱，打回！）
5. **事实锚点**：是否至少包含了两个硬核数据、真实机构或专业机制词汇？（如果是纯水文，打回！）
6. **对立杠点**：结尾是否抛出了非黑即白的站队问题？（如果是“你怎么看”或“让我们期待”，打回！）

## ⚙️ 第四阶段：洗稿重铸与终极输出
1. 依据第三阶段中你自己提出的修改意见，在脑内彻底重设比喻、补足数据、强化情绪和互动倒钩。
2. **只有在草稿完美通过红队检验后，才正式向用户进行自然语言输出。**
3. 输出结果应包含：
   - 最终呈现给读者的爆款正文。
   - 文章末尾必须附带更新过后的 `【🚨 红队自审报告】`（向用户证明你已经完成了 4D 矩阵与打假三板斧的检验）。
