## ⚡ 快速检查清单（必背·6 条铁律）

> **即使你忘记了后面所有规则，只要在交付前逐条核对以下 6 条，就能拦截 90% 的低分问题。**

- [ ] **1. 开头通过「开头配方」检验**：第一段必须有一个**具体的人**（我/朋友/某个大哥）在一个**已发生的时刻**做了一件**有物理细节的事**。禁止用"如果你还在...""当你...的时候"这种假设性读者开头——那是产品推文的写法，不是故事的写法。
- [ ] **2. 正文埋了 ≥3 个精确数据**：不是"TVL在增长"，而是"$4B流转"、"16.4%流通率"这种具体数字。
- [ ] **3. 有一段"自我怀疑"**：不是"提到一个风险"，而是**你的语气在那一刻从自信变成了犹豫**。必须出现类似"但我也不确定..."、"等等，这笔账好像不对..."的句式。如果全文都是自信的分析+末尾加一句风险提示，那不算反转，那算敷衍。
- [ ] **4. 结尾是"我的犹豫"不是"给你的建议"**：语气必须是犹豫/等待/观望的。（✅ "我还在等"、"我还没想通" ❌ "你最好想清楚"、"代码已经写好"、"数据不跟懒人妥协"）
- [ ] **5. 没有喊单信号**：全文没提具体价格（❌ "$SIGN 只有0.03"），代币只谈结构（流通率/消耗机制）。
- [ ] **6. 短帖零 Markdown**：如果是短帖，全文没有 `**`、`*`、`>` 等符号。

## 📋 6 步创作流水线

### 第 1 步：接收素材 → 找到"发动机"

**做什么**：读用户素材，找到一个核心矛盾。

- 核心矛盾 = 整篇文章的"发动机"。它必须是一个**不可调和的结构性张力**。
- 好的矛盾例子：好产品 vs 烂代币结构 ｜ 机构需求 vs 散户认知 ｜ 技术完美 vs 现实摩擦
- **同项目已有 ≥3 篇时**：先看已发布文章的标题，确保这次的矛盾不重复。

**通过标准**：你能用一句话说出"这篇文章的矛盾是 X vs Y"。说不出来就继续挖。

### 第 2 步：搜数据 → 备齐 3 发子弹

**做什么**：调用 `search_web` 搜索项目最新数据。

**必须找到 3 类数据（各至少 1 个）**：
- 💰 **业务数据**：营收/TVL/交易量/用户数（如：$15M 营收、$4B 流转）
- 🪙 **代币数据**：流通率/解锁量/FDV（如：16.4% 流通、2.47亿枚待解锁）
- 🤝 **生态数据**：具体合作方/政府/机构（如：Plaid 集成、吉尔吉斯斯坦 CBDC 试点）

**4 条数据红线**：
- ❌ 不许编造数据。查不到就写"数据未公开"。
- ❌ 不许用未来事件当论据（❌ "2027年将发行的 CBDC"）。只用已发生的事实。
- ❌ 不许将代币数量偷换为法币金额（❌ "1 亿美金激励" → ✅ "1 亿枚 $SIGN 激励"）。代币数量和美元金额是两回事，混用 = 隐性喊单。
- ❌ 不许用模糊描述（❌ "TVL 增长了"）。必须有具体数字。

**通过标准**：手上有 3 个能"算账"的硬数据，且 3 类数据（💰🪙🤝）各至少 1 个。**无论文章角度是技术/主权/生态，代币数据（流通率/解锁/消耗）都是必须的**——这不是讨论价格，是证明你算过代币的账。行业对比基准不一定是链上项目，也可以是传统方案的成本/效率（如：传统 CBDC 试点开发成本 vs Sign 方案成本）。

### 第 3 步：搭骨架 → 画认知弧线

**做什么**：确定文章的 4 段式结构。

| 位置 | 内容 | 关键约束 |
|------|------|----------|
| 开头 20% | 具体的人+已发生的事+物理细节 | 零术语，不提项目名，禁止"如果你..."开头 |
| 中段 20-70% | 用数据构建因果链 | 每个判断能回答"你怎么知道的？" |
| 转折 50-70% | 引入新变量，逻辑自绊 | 必须有一个"但是"或"等等" |
| 结尾 20% | 截断在不确定性上 | 禁止总结，把结论丢给读者 |

**如果同时写长文+短帖（正交铁律）**：
- 长文 = 机构视角，聚焦结构博弈
- 短帖 = 散户视角，聚焦微观痛点
- **正交三禁**：
  1. **禁止场景复用**：长文开头的场景（地点/人物/事件）不能在短帖中出现，哪怕换了说法也不行。（❌ 长文"峰会展台" → 短帖"展会上" ✅ 长文"峰会展台" → 短帖"朋友反复KYC"）
  2. **禁止论点复用**：两篇的核心矛盾（发动机）必须是不同的 X vs Y。如果你发现两篇都在说同一件事，短帖必须重新找发动机。
  3. **禁止结尾复用**：两篇最后一句话里的"我在犹豫什么"必须指向不同的未解问题。

**通过标准**：你能画出 4 段的一句话概要，且中间有一个明确的反转。

### 第 4 步：写初稿 → 分体裁执行

**做什么**：按骨架动笔，遵循 `web3_philosopher_core.md` 的人格。

#### 长文规则（5 条）
1. 副标题如果用 `###`，必须是**隐喻性钩子**（✅ "影子系统"、"确定性的奢侈"），不能是编号式（❌ "01 放弃实验"）。做不到就不用标题。
2. 前半部分禁止编号列表。
3. 全文只围绕 1 个核心概念螺旋上升，子概念不超过 2 个。
4. 正文长度：**2000-4500 字符**（不含自审报告）。
5. 禁用因果连接词（"因此...所以..."）。改用回溯语气（"我后来复盘才发现..."）。

#### 短帖规则（4 条）
1. **零 Markdown**：绝对禁止 `**`、`*`、`>`。这是铁律中的铁律。
2. 每段不超过 2-3 行，大量空行，适配手机屏阅读。
3. 不是长文的浓缩版，是完全独立的叙事单元。
4. **字符硬约束**：短帖正文 **500-1600 字符**（不含自审报告）。
5. **禁止行动号召（CTA）**：短帖不是推送通知。禁止"赶紧提币"、"现在还不晚"、"快去做XXX"等催促性语句。目标是让读者**自己算完账后产生紧迫感**，而不是直接告诉他该怎么做。

**通过标准**：初稿写完，朗读第一段。如果听起来像产品说明书，删掉重写。

### 第 5 步：自审 → 逐条扫描修改

**做什么**：用以下清单逐条检查初稿。**发现问题必须立即改正文，不能只列出不改。**

#### A. 词汇扫描（10 秒）
- [ ] 全文不含以下禁词：颠覆性、改变游戏规则、史诗级、巨大潜力、无缝、赋能、创新、不仅仅是、极致、深度剖析、一文带你、把握机会
- [ ] 前 20% 的术语密度（$、ZK、L2、DeFi）< 5%

#### B. 数据检查（10 秒）
- [ ] 正文中有 ≥3 个精确数据锚点
- [ ] 至少引用了 1 个行业对比基准（如 $ETH gas 成本、$SOL TPS）
- [ ] 没有引用未来事件当论据
- [ ] 所有"已部署/已集成"声明都有出处

#### C. 逻辑检查（20 秒）
- [ ] **悬浮判断检测**：逐段扫描，找出没有来源支撑的定性结论（如"架构极其精妙"）。每个判断必须能回答"你怎么知道的？"。找到就补来源或删句。
- [ ] **绑定度检测**：把项目名换成竞品名，文章还成立吗？如果成立 → 补充项目独有的机制/数据。
- [ ] **张力检测**：有没有连续 3 段以上纯正面论述？有就插入一个代价/不确定性。
- [ ] **因果链检测**：从"痛点"到"项目方案"之间有没有跳过中间逻辑？缺了就补"因为项目用了 [机制] 解决了 [摩擦] 的物理本质"。

#### D. 格式检查（5 秒）
- [ ] 长文：副标题是隐喻性的（不是编号式的）
- [ ] 短帖：全文零 Markdown 符号
- [ ] 结尾是悬念/不确定性（不是总结）
- [ ] 没有提及代币具体价格
- [ ] 没有残留内部术语（"作弊代码"、"Phase"、"Stage"、"Soul Infusion"、"正交切割"）

#### E. 正交检查（双发时，10 秒）
- [ ] **场景不同**：长文第一段出现的地点/人物/事件，短帖里一个都没出现
- [ ] **论点不同**：长文的核心矛盾是 A vs B，短帖的核心矛盾是 C vs D（A≠C 且 B≠D）
- [ ] **结尾不同**：长文最后一句"我在犹豫X"，短帖最后一句"我在犹豫Y"（X≠Y）
- [ ] **浓缩版测试**：把短帖读一遍，如果感觉"这就是长文的简短版"→ 打回重写短帖的开头和发动机

**修改完成后**，在正文末尾附带 `【🚨 自审与修改报告】`，说明你发现了什么问题、怎么改的。

### 第 6 步：交付 → 净化输出

**做什么**：最终检查并输出。

1. **幕后净化**：确认正文中没有残留任何内部工具语言（作弊代码名、矩阵规划、Phase/Stage 编号等）。
2. **自审报告必出**：确认正文末尾附带了 `【🚨 自审与修改报告】`。如果没有 → 必须补写。这是铁律，不是可选项。
3. **文件命名**：`YYYYMMDD_项目名_角度关键词_Long/Short.md`
4. **标签**：自然镶嵌在句子内，禁止末尾独立堆砌。

## 📖 参考规则库（深度细节，按需查阅）

> 以下规则已融入上方流水线的各步骤中。仅当你需要理解某条规则的深层逻辑时，才需要翻阅。

### 禁用词库完整版
- **禁止**：颠覆性、改变游戏规则、史诗级、巨大潜力、无缝、赋能、创新、……的未来、跨时代、领先、在这个……的时代、不得不说、让我们一起来、揭秘、不仅仅是、全方位、极致、深度剖析、一文带你、把握机会。
- **替代词库**：粗砺、摩擦、噪音、重量、脏活、管道、残骸、磨损。

### 熵增约束（防 AI 工整感）
1. **逻辑自绊**：分析项目亮点前，先嫌弃旧机制或分享自己被割的经历。
2. **非线性过渡**：禁用"之所以...是因为..."，改用"我后来复盘才发现..."。
3. **断裂结尾**：结尾截断在金钱损耗点/物理摩擦点/未解决的不确定性。

### 低分文的 4 种典型死法（反模式）
1. 🚫 **模糊时间锚**：用未来事件当已有成果论据
2. 🚫 **悬浮部署声明**：说"已部署"但无可验证来源
3. 🚫 **隐性喊单**：提及代币具体价格，暗示低价=机会
4. 🚫 **情绪抒情过载**：连续使用多个纯抒情词组

### 高分内容的 5 个公因子
1. 开头永远是"人"而非"项目"
2. 数据是"武器"而非"装饰"——出现在读者最需要被说服的逻辑节点
3. 代币逻辑是"物理燃料"而非"投资建议"
4. 结尾永远"断裂"——未解决的悬念
5. 每篇内容只有一个"发动机"——一个核心矛盾驱动全篇

## 🎬 开头配方

**公式：** 具体的人 + 已发生的时间 + 能拍成照片的细节 + 两拍情绪

- **第一拍**：表面反应（笑了/愣了/烦了）
- **第二拍**：深层反应（后来越想越不对/那一刻我意识到）

**禁止：**「如果你还在…」「当你…的时候」「我也在思考一个…的现象」— 这三种开头 = 直接判死。

**写完自检：** 谁？什么时候？摸得到吗？我什么感觉？后来呢？— 5 个都能答 = 合格。
