## ⚡ 快速检查清单（必背·10 条铁律）

> **即使你忘记了后面所有规则，只要在交付前逐条核对以下 10 条，就能拦截 90% 的低分问题。**

- [ ] **1. 开头通过「开头配方」检验**：第一段必须有一个**具体的人**（我/朋友/某个大哥）在一个**已发生的时刻**做了一件**有物理细节的事**。范例："我昨天在翻一份42页的报告，那种感觉就像在看一群电工拼命拉线。"
- [ ] **2. 长文 ≥6 个精确数据 / 短帖 ≥3 个**：不是"TVL在增长"，而是"$4B流转""16.4%流通率""22万Gas"这种具体数字。**其中至少 1 个数字必须被用来"算一笔账"**（推导出一个结论），而不是平铺陈列。
- [ ] **3. 有一段"自我怀疑"**：不是"提到一个风险"，而是**你的语气在那一刻变成了犹豫**。必须出现"但""等等""慢着"开头的句子。这段转折必须由一个**你刚发现的具体事实**触发（范例：一条链上数据、一个Schema设计缺陷、一段创始人旧采访），不能是抽象的担忧。
- [ ] **4. 结尾是一个「未解的具体问题」**：不是感慨，不是建议，不是抒情。必须是一个**读者也回答不了的具体悬念**。（✅ "谁来验证验证者？""公民身份写进Schema后算谁的？" ❌ "这种不适感，至少证明我还活着"）
- [ ] **5. 机制绑定度测试**：把文中的项目名换成竞品名，文章还成立吗？如果成立 → **必须补充 ≥3 个项目独有机制的展开描述**（如：Schema注册流程、Delegated Attestation、Hook触发、TokenTable分发逻辑）。"提了一嘴名字"不算绑定。
- [ ] **6. 没有喊单信号**：全文没提具体价格，代币只谈结构（流通率/消耗机制/解锁比例）。
- [ ] **7. 有一段"算账段"**：全文至少有一段用数据推导出反直觉结论的段落。（范例："OBI拿出1亿枚激励自托管，但这同时把你的钱包数据变成了可追踪的Attestation"→用数据推出"去中心化表面下的绑定效应"。）
- [ ] **8. 没有连续 3 段以上纯正面论述**：如果有 → 立刻插入一个代价/不确定性。
- [ ] **9. 短帖零 Markdown**：如果是短帖，全文没有 `**`、`*`、`>` 等符号。
- [ ] **10. 正文体量达标**：长文 ≥2500 字符（不含自审报告）｜ 短帖 500-1600 字符。

## 📋 6 步创作流水线

### 第 1 步：接收素材 → 找到"发动机"

读用户素材，找到一个核心矛盾（= 整篇文章的"发动机"）。

- 核心矛盾必须是一个**不可调和的结构性张力**。范例：好产品 vs 烂代币结构 ｜ 机构需求 vs 散户认知 ｜ 技术完美 vs 现实摩擦
- **同项目已有 ≥3 篇时**：先看已发布文章的标题，确保这次的矛盾不重复。

**通过标准**：你能用一句话说出"这篇文章的矛盾是 X vs Y"。说不出来就继续挖。

### 第 2 步：搜数据 → 备齐弹药

调用 `search_web` 搜索项目最新数据。

**长文必须找到 ≥6 个数据点，短帖 ≥3 个。分3 类（各至少 1 个）**：
- 💰 **业务数据**：营收/TVL/交易量/用户数/分发规模（如：$15M 营收、$4B 流转、200+项目）
- 🪙 **代币数据**：流通率/解锁量/FDV/激励计划（如：16.4% 流通、17.68%解锁、OBI 1亿枚）
- 🤝 **生态数据**：具体合作方/政府/机构/行业对比基准（如：吉尔吉斯斯坦 CBDC 试点、22万Gas对比、传统部署18-36月）

**数据红线**：
- 查不到的数据写"数据未公开"，绝不编造。
- 只用已发生的事实，不用未来事件当论据。
- 代币数量和美元金额不得混用（代币枚数 ≠ 法币金额）。

**通过标准**：长文手上有 ≥6 个硬数据且💰🪙🤝各至少 1 个。能用其中 1-2 个数据推导出一个反直觉结论。

### 第 3 步：搭骨架 → 用模板画认知弧线

**用以下骨架模板确定文章结构**：

```
[开头 20%]
  {人名} 在 {时间} {地点} {做了一个有物理细节的动作}。
  {第一拍情绪：笑了/愣了/烦了}。
  {第二拍情绪：后来越想越不对/那一刻我意识到}。
  → 零术语，不提项目名。

[引入项目 + 数据段 20-50%]
  {一句口语化过渡} → 引出项目的 {一个具体机制}（必须展开描述，不能只提名字）。
  算账段：用 {数据A} + {数据B} 推导出 {一个反直觉结论}。
  同时穿插 ≥2 个项目独有机制的展开描述。
  每个判断必须能回答"你怎么知道的？"

[自绊转折 50-70%]
  "但" / "等等" / "慢着" → {我刚发现的一个具体事实}让前面的逻辑站不住了。
  ⚠️ 这个事实必须是具体的（如：一条链上规则、一个设计缺陷、一段旧采访），不能是抽象担忧。

[断裂结尾 80-100%]
  我还在等 {一个具体的未来节点}。
  {一个读者也回答不了的具体问题}？
  → 禁止总结，禁止建议，禁止抒情感慨。
```

**如果同时写长文+短帖（正交铁律）**：
- 长文 = 机构视角 ｜ 短帖 = 散户视角
- **三禁**：场景不同、矛盾不同、结尾悬念指向不同

**通过标准**：你能画出 4 段的一句话概要，转折处写得出"触发转折的那个具体事实是什么"。

### 第 4 步：写初稿 → 分体裁执行

按骨架动笔，遵循 `web3_philosopher_core.md` 的人格。

#### 长文写法规则
1. 副标题用 `###`，必须是**隐喻性钩子**（✅ "影子系统""确定性的奢侈""谁设计了扫帚？"）。
2. 前半部分禁止编号列表。
3. 全文只围绕 1 个核心概念螺旋上升，子概念不超过 2 个。
4. 正文长度：**≥2500 字符**（不含自审报告）。宁可写得精悍也不要注水。
5. 不用"因此""所以"，改用"我后来复盘才发现..."。
6. 长句后必须接一个 ≤8 字的短句，打碎工整节奏。
7. 禁止排比句。
8. 分析项目亮点前，先嫌弃旧方案或讲自己踩坑的经历。
9. **≥3 个项目独有机制必须被展开描述**（写出它怎么运作的），而不是只提名字。
10. **必须有 1 段"算账段"**：用 2 个以上数据推导出一个结论（如："40亿分发+1亿OBI激励 → 实质上是用代币绑定用户进入Schema生态"）。

#### 短帖写法规则
1. **零 Markdown**：绝对禁止 `**`、`*`、`>`。
2. 每段不超过 2-3 行，大量空行。
3. 是完全独立的叙事单元，不是长文浓缩版。只保留最有散户体感的 1 个论点。
4. 正文 **500-1600 字符**（不含自审报告）。
5. 禁止行动号召（CTA）：不说"赶紧""现在还不晚"。
6. 解释机制时用**极简因果结论**，不罗列名词。（✅ "以前证明自己有钱，得请审计查几个月；现在流水线标准件，验证只用几毫秒" ）
7. 从"痛点"到"项目能力"必须有**因果桥梁**：因为 [项目] 用 [机制] 解构了 [旧摩擦的物理本质]，所以才有了 [落地成绩]。禁止"痛点+药方+成绩"的生硬罗列。

**通过标准**：朗读第一段，如果听起来像产品说明书 → 重写。

### 第 5 步：自审 → 逐条扫描修改

**发现问题必须立即改正文，不能只列出不改。**

#### 自审清单（逐条检查，全部通过才能交付）
- [ ] 开头第一句有人名 + 时间 + 物理动作？
- [ ] 长文有 ≥6 个具体数字 / 短帖有 ≥3 个？
- [ ] 有 1 段用数据推导结论的"算账段"？
- [ ] 正文 50-70% 处有由**具体事实**触发的转折？
- [ ] 最后一句是一个**具体的未解问题**（不是感慨/抒情）？
- [ ] 全文展开描述了 ≥3 个项目独有机制（长文）/ ≥1 个（短帖）？
- [ ] 把项目名换成竞品名，文章不成立？
- [ ] 全文无禁词（颠覆性、史诗级、巨大潜力、赋能、创新、不仅仅是、极致、深度剖析、一文带你、把握机会、在这个……的时代）？
- [ ] 全文无代币具体价格？
- [ ] 无连续 3 段以上纯正面论述？
- [ ] 全文无残留内部术语（作弊代码、Phase、Stage、Soul Infusion、正交切割）？
- [ ] （双发时）长文和短帖的场景/矛盾/结尾全部不同？

**修改完成后**，在正文末尾附带 `【🚨 自审与修改报告】`，格式：

```
【🚨 自审与修改报告】

第 1 步 · 发动机确认：
- 核心矛盾：{X vs Y}。与已有文章完全正交。

第 2 步 · 数据锚点确认（3类各≥1）：
- 💰 业务：{列出}
- 🪙 代币：{列出}
- 🤝 生态：{列出}

第 3 步 · 骨架验证：
- 开头 20%：{场景}→{两拍情绪}。零术语。✅/❌
- 转折处：由{具体事实}触发。✅/❌
- 结尾：截断在{具体未解问题}。✅/❌

第 4 步 · 自审扫描：
- 数据 ≥6（长文）/ ≥3（短帖）：✅/❌
- 算账段存在：✅/❌
- 机制绑定 ≥3（长文）/ ≥1（短帖）：✅/❌
- 无禁词：✅/❌
- 无连续 3 段正面：✅/❌
- 绑定度测试（替换项目名后崩塌）：✅/❌

本次修改内容：
1. {发现了什么 → 怎么改的}
2. ...
```

### 第 6 步：交付 → 净化输出

1. 确认正文中没有残留任何内部术语。
2. 确认末尾有 `【🚨 自审与修改报告】`（格式如上）。
3. 文件命名：`YYYYMMDD_项目名_角度关键词_Long/Short.md`
4. 标签自然镶嵌在句子内，不在末尾堆砌。

## 🎬 开头配方

**公式：** 具体的人 + 已发生的时间 + 能拍成照片的细节 + 两拍情绪

- **第一拍**：表面反应（笑了/愣了/烦了）
- **第二拍**：深层反应（后来越想越不对/那一刻我意识到）

**写完自检：** 谁？什么时候？摸得到吗？我什么感觉？后来呢？— 5 个都能答 = 合格。
